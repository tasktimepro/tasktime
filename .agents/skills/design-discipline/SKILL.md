# Design Discipline

Use for any UI, UX, layout, component, form, navigation, dialog, state, accessibility, or visual-refinement task.

1. Identify the user's task, primary action, essential information, secondary actions, and failure states.
2. Inspect existing TaskTime Pro components, page patterns, tokens, and Radix primitives before proposing anything new.
3. Choose the most familiar fitting pattern and follow `rules/design-discipline.md`.
4. Check hierarchy, action placement, microcopy, responsive composition, keyboard behavior, accessible names, focus, contrast, and non-color cues.
5. Cover loading, empty, error, success, disabled, selected, and destructive states where relevant.
6. Verify the implementation at meaningful viewport sizes and add behavior/accessibility tests proportionate to the change.

If a custom interaction is materially more unusual than the standard pattern, explain the standard option, the proposed variant, and the reason, then obtain approval before implementing it.
