TaskTime now has two new planning features inside each project: a Kanban task view and a Notes tab.

This is meant to solve a simple problem. Most freelancers end up splitting planning across a task app, a notes app, and a timer. That creates extra admin and makes it easier to miss work when it is time to invoice.

## Kanban view for project tasks

You can now switch a project between List and Kanban view.

In Kanban:

- Parent tasks become columns.
- Subtasks become cards.
- You can drag columns to reorder the project.
- You can drag cards within a column or move them to another parent task.
- Recurring and archived tasks stay available without cluttering the active board.

The important part is that this is still the same task data. Your timers, time entries, billable state, and invoice data stay attached to the task while you reorganize the work.

TaskTime also remembers the task view per project, so if one client works better as a Kanban board and another works better as a list, you do not have to keep switching it back.

## Built-in project notes

Every project now has a Notes tab next to Tasks.

You can use it for:

- Quick project plans
- Checklists
- Meeting notes
- Links
- Headings and bullet lists
- Client reminders

Notes save locally right away so writing stays fast. If you use Google Drive sync, you can save notes to the cloud and keep them synced without moving your project details into a separate document tool.

## Why this matters

Planning works better when the task board, the notes, the timer, and the invoice all live in the same place.

That means:

- Less context switching
- Fewer missed billable tasks
- Cleaner handoff from planning to time tracking
- Clearer project status when you come back later

This update gives TaskTime a better task management workflow without changing what makes it useful in the first place.

## What stays the same

TaskTime is still:

- Local-first
- Private by default
- Usable offline
- Optional sync, not account-first
- Built for freelancers and solo work

Kanban and project notes add structure, but they do not turn the app into a heavy project management system.

## Try it

Open any project in TaskTime, switch the task view to Kanban, move a few subtasks into place, and use the Notes tab to keep the plan beside the work.
